Drop the `changelog` bundled practice. Fragment-based changelog is one
specific technique among several (auto-generated, manually-maintained,
release-notes-only, none); the bundled catalog should only ship
universal practices, so changelog mechanism is now overlay territory.
The `commits` practice and `core-rules` are softened to describe the
common changelog mechanisms without prescribing fragments. Bundled
catalog drops from 6 to 5 entries.
